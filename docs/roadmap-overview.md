# FloodRush Roadmap Overview

This document translates the product vision into an implementation-oriented view.

## Phase 1: foundation
- Shared contracts and domain concepts
- Core gameplay model and scoring rules
- Device identity and authenticated server communication
- Unit test projects and coverage reporting for the core game and server modules

## Phase 2: playable vertical slice
- MAUI shell and welcome page
- One or more playable levels
- Local persistence for progress and settings

## Phase 3: connected gameplay
- Level release and download flow
- Online synchronization of profile, settings, and scores
- Top-score viewing and comparison

## Phase 4: expansion
- More level content
- Deeper progression tuning
- Quality, balancing, and platform hardening
- Maintain at least 80% code coverage for the core projects as new logic is added
